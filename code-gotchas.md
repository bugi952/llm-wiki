# Code Gotchas -- LLM Wiki

Claude 코딩에서 반복되는 실수. /review-code 스킬이 이 기준으로 채점한다.

## 채점 기준

| 기준 | 가중치 | 설명 | 기준 미달 |
|------|--------|------|-----------|
| **설계 충실도** | 35% | docs/에 명시된 설계를 그대로 따랐는가? 임의 확장/변경 없는가? | 3점 미만 -> 설계 재확인 후 수정 |
| **최소 구현** | 25% | 요구한 것만 구현했는가? 불필요 추상화/레이어/유틸 없는가? | 3점 미만 -> 불필요 코드 제거 |
| **완성도** | 25% | stub/TODO 없이 실제 동작? 에러 핸들링 실제 작동? | 3점 미만 -> 미완성 부분 완성 |
| **수락 조건 충족** | 15% | docs/testing.md Phase 수락 조건 전부 통과? | 2점 미만 -> 수락 조건 재실행 |

각 5점 만점. **가중 평균 3.5점 미만이면 건재에게 전달하지 않고 수정.**

## 퀵체크 (기능 완성 시마다 셀프)
1. docs/00, 01 읽었는가? 설계와 다르게 구현한 부분은?
2. "나중에 쓸" 코드를 만들지 않았는가?
3. TODO/FIXME/placeholder가 남아있지 않은가?
4. 에러 핸들링이 실제로 동작하는가? (bare except 금지)
5. 수락 조건 테스트를 실행했는가?
